# 테스트 질문 시나리오

이 문서는 `compact_manufacturing_service`의 주요 기능을 영역별로 점검하기 위한 테스트 질문 모음입니다.

권장 테스트 순서:
1. 기본 조회가 정상 동작하는지 확인
2. 도메인 조건 추출이 자연스럽게 되는지 확인
3. 첫 조회 이후 파라미터 승계가 되는지 확인
4. `current_data` 기반 pandas 후속 분석이 유연하게 되는지 확인
5. 다른 데이터 주제로 넘어갈 때도 주요 조건이 이어지는지 확인

---

## 1. 기본 조회 테스트
목적:
- 생산 / 목표 / 불량 / 설비 / WIP 조회가 모두 동작하는지 확인
- 날짜 해석과 기본 응답이 자연스러운지 확인

질문:
- 오늘 생산량 보여줘
- 오늘 목표 보여줘
- 오늘 불량 보여줘
- 오늘 설비 가동률 보여줘
- 오늘 WIP 보여줘
- 어제 생산량 보여줘
- 어제 불량 보여줘

확인 포인트:
- 날짜가 올바르게 해석되는지
- 표가 생성되는지
- 응답 텍스트가 현재 결과 테이블 기준으로 설명되는지
- 추출된 필터 조건에 날짜가 보이는지

---

## 2. 공정군 도메인 인식 테스트
목적:
- 제조 공정군 용어가 질문에서 잘 추출되는지 확인
- 공정군 질문이 실제 공정 단계 데이터로 자연스럽게 연결되는지 확인

질문:
- 오늘 ASSY_PREP 공정 생산량 보여줘
- 오늘 DIE_ATTACH 공정 생산량 보여줘
- 오늘 WIRE_BOND 공정 불량 보여줘
- 오늘 FLIP_CHIP 공정 불량 보여줘
- 오늘 MOLD 공정 생산량 보여줘
- 오늘 SINGULATION 공정 생산량 보여줘
- 오늘 TEST 공정 생산량 보여줘
- 오늘 PACK_OUT 공정 WIP 보여줘

확인 포인트:
- `process_name`이 공정군 기준으로 잘 추출되는지
- 해당 공정군과 연결된 실제 공정 단계가 결과에 포함되는지
- 후속 질문 없이도 공정군 필터가 자연스럽게 반영되는지

---

## 3. 제품 / MODE / DEN / TECH 조건 추출 테스트
목적:
- 제품 / MODE / DEN / TECH / MCP_NO 조건이 질문에서 잘 추출되는지 확인
- 도메인 지식 기반 필터가 정상 동작하는지 확인

질문:
- 오늘 DDR5_6400 생산량 보여줘
- 오늘 DDR5_5600 생산량 보여줘
- 오늘 LPDDR5X_8533 생산량 보여줘
- 오늘 UFS3.1 생산량 보여줘
- 오늘 PMIC 생산량 보여줘
- 오늘 CIS 생산량 보여줘
- 오늘 64Gb 제품 생산량 보여줘
- 오늘 FC 기술 제품 생산량 보여줘
- 오늘 WB 기술 제품 불량 보여줘
- 오늘 MCP-LP5X-064G-4H 제품 불량 보여줘

확인 포인트:
- MODE / DEN / TECH / MCP_NO가 맞게 추출되는지
- 결과가 해당 조건과 일치하는지
- 추출된 필터 조건이 보기 쉽게 표시되는지

---

## 4. 설비 데이터 테스트
목적:
- 설비 데이터가 실제 현장 데이터처럼 자연스럽게 보이는지 확인
- 설비명 / 설비ID / 공정 / 라인 / 가동률 / 비가동 사유가 읽기 쉬운지 확인

질문:
- 오늘 설비 가동률 보여줘
- 오늘 DIE_ATTACH 설비 상태 보여줘
- 오늘 WIRE_BOND 설비 상태 보여줘
- 오늘 TEST 설비 상태 보여줘
- 가동률 낮은 순으로 정렬해줘
- 비가동 사유가 있는 설비만 보여줘

확인 포인트:
- 설비ID와 설비명이 자연스러운지
- 가동률과 비가동 사유가 표에 잘 보이는지
- 후속 정렬 요청이 정상 반영되는지

---

## 5. 불량 데이터 테스트
목적:
- 불량 데이터와 주요 불량 유형 분류가 자연스러운지 확인
- 공정별 / 제품별 후속 분석이 잘 되는지 확인

질문:
- 오늘 불량 보여줘
- 오늘 DIE_ATTACH 불량 보여줘
- 오늘 WIRE_BOND 불량 보여줘
- 오늘 FLIP_CHIP 불량 보여줘
- 오늘 TEST 불량 보여줘
- 불량률 높은 순으로 정렬해줘
- 주요 불량 유형별로 그룹화해줘
- 공정군 기준 평균 불량률 보여줘

확인 포인트:
- 불량률 / 불량수량 / 주요불량유형이 자연스럽게 보이는지
- 정렬과 그룹화 요청이 pandas 코드로 잘 풀리는지

---

## 6. WIP 데이터 테스트
목적:
- 재공수량 / 홀드 / 대기 상태 데이터가 자연스럽게 보이는지 확인
- 상태 기반 후속 분석이 잘 되는지 확인

질문:
- 오늘 WIP 보여줘
- 오늘 TEST 공정 WIP 보여줘
- 오늘 LPDDR5X_8533 WIP 보여줘
- HOLD 상태만 보여줘
- 재공수량 상위 10개만 보여줘
- 공정군별로 그룹화해줘
- 라인별로 그룹화해줘

확인 포인트:
- WIP 상태 흐름이 자연스러운지
- 재공수량 컬럼 표시가 일관적인지
- 필터 후 재분석이 정상 동작하는지

---

## 7. 파라미터 승계 테스트
목적:
- 첫 조회에서 잡힌 날짜 / 제품 / 공정 조건이 다음 질문에 자연스럽게 이어지는지 확인

질문:
- 오늘 LPDDR5X_8533 생산량 보여줘
- 불량 보여줘
- WIP 보여줘

확인 포인트:
- `오늘`, `LPDDR5X_8533` 조건이 후속 질문에도 유지되는지
- 데이터 주제가 바뀌어도 주요 조건이 이어지는지

---

## 8. 단순 후속 pandas 분석 테스트
목적:
- 현재 결과 테이블에 대해 단순 필터 / 정렬 / 상위 N이 잘 되는지 확인

질문:
- 오늘 생산량 보여줘
- 상위 5개만 보여줘
- DDR5_5600만 보여줘
- 생산량 높은 순으로 정렬해줘
- 라인별로 그룹화해줘

확인 포인트:
- 후속 질문이 새 조회가 아니라 현재 테이블 분석으로 처리되는지
- 생성된 pandas 코드가 질문 의도와 맞는지

---

## 9. 복합 후속 pandas 분석 테스트
목적:
- 그룹화 + 정렬 + 상위 N 같은 복합 요청이 자연스럽게 처리되는지 확인

질문:
- 오늘 생산량 보여줘
- mode, den, tech별로 그룹화해주고 그룹별로 상위 3개씩만 보여줘
- 공정군별로 그룹화한 뒤 생산량 높은 순으로 상위 4개만 보여줘
- LPDDR5X_8533만 남기고 공정군별로 요약해줘

확인 포인트:
- 복합 요청이 정형 규칙이 아니라 pandas 코드로 자연스럽게 풀리는지
- 결과 표와 텍스트 설명이 같은 데이터 기준인지

---

## 10. 없는 컬럼 요청 안내 테스트
목적:
- 현재 결과에 없는 컬럼을 요청했을 때 명확하게 안내하는지 확인

질문:
- 오늘 목표 보여줘
- 공정군별로 그룹화해줘
- MODE별로 그룹화해줘

추가 질문:
- 오늘 생산량 보여줘
- 담당자별로 정리해줘

확인 포인트:
- `현재 결과 테이블에 없습니다` 형태로 안내하는지
- 현재 사용 가능한 주요 컬럼도 함께 보여주는지

---

## 11. 주제 전환 테스트
목적:
- 생산 -> 불량 -> 설비 -> WIP로 넘어가도 주요 파라미터가 이어지는지 확인

질문:
- 오늘 TEST 공정 생산량 보여줘
- 불량 보여줘
- 설비 상태도 보여줘
- WIP도 보여줘

확인 포인트:
- `오늘`, `TEST 공정` 조건이 유지되는지
- 완전히 다른 도구로 넘어가도 흐름이 자연스러운지

---

질문:

- 상위 5개만 보여줘

- 오늘 WIRE_BOND 공정 불량 보여줘
- 오늘 TEST 공정 WIP 보여줘
- 설비 상태도 보여줘
