# 테스트 질문 시나리오

이 문서는 `compact_manufacturing_service`의 핵심 기능을 전반적으로 점검하기 위한 영역별 테스트 질문 모음입니다.

권장 테스트 순서:
1. 기본 조회가 정상 동작하는지 확인
2. 도메인 조건 추출이 맞는지 확인
3. 첫 조회 이후 파라미터 승계가 되는지 확인
4. `current_data` 기반 pandas 후속 분석이 유연하게 동작하는지 확인
5. 공정군, 제품군, 설비, WIP, 불량 데이터가 서로 자연스럽게 보이는지 확인

---

## 1. 기본 조회 테스트

목적:
- 생산 / 목표 / 불량 / 설비 / WIP 조회가 모두 동작하는지 확인
- 날짜 해석과 기본 응답이 자연스러운지 확인

질문:
- 오늘 생산량 보여줘
- 오늘 목표 보여줘
- 오늘 불량 보여줘
- 오늘 설비 가동률 보여줘
- 오늘 WIP 보여줘
- 어제 생산량 보여줘
- 어제 불량 보여줘

확인 포인트:
- 날짜가 올바르게 해석되는지
- 표가 생성되는지
- 응답 텍스트가 현재 결과 테이블 기준으로 설명되는지
- 추출된 필터 조건에 날짜가 보이는지

---

## 2. 공정군 도메인 인식 테스트

목적:
- 추가된 제조 도메인 공정군이 질문에서 잘 추출되는지 확인
- 공정군 질문이 실제 공정 스텝 데이터로 확장되는지 확인

질문:
- 오늘 ASSY_PREP 공정 생산량 보여줘
- 오늘 DIE_ATTACH 공정 생산량 보여줘
- 오늘 WIRE_BOND 공정 불량 보여줘
- 오늘 FLIP_CHIP 공정 불량 보여줘
- 오늘 MOLD 공정 생산량 보여줘
- 오늘 SINGULATION 공정 생산량 보여줘
- 오늘 TEST 공정 생산량 보여줘
- 오늘 PACK_OUT 공정 WIP 보여줘

확인 포인트:
- `process_name`이 공정군 기준으로 잘 잡히는지
- 공정군에 속한 실제 공정 스텝들이 결과에 포함되는지
- 후속 질문 없이도 공정군 필터가 자연스럽게 반영되는지

---

## 3. 제품 / MODE / DEN / TECH 조건 추출 테스트

목적:
- 제품 관련 도메인 조건이 정확히 추출되는지 확인
- MODE / DEN / TECH / MCP_NO 조건이 필터에 반영되는지 확인

질문:
- 오늘 DDR5_6400 생산량 보여줘
- 오늘 DDR5_5600 생산량 보여줘
- 오늘 LPDDR5X_8533 생산량 보여줘
- 오늘 UFS3.1 생산량 보여줘
- 오늘 PMIC 생산량 보여줘
- 오늘 CIS 생산량 보여줘
- 오늘 64Gb 제품 생산량 보여줘
- 오늘 FC 기술 제품 생산량 보여줘
- 오늘 WB 기술 제품 불량 보여줘
- 오늘 MCP-LP5X-064G-4H 제품 불량 보여줘

확인 포인트:
- MODE / DEN / TECH / MCP_NO가 맞게 잡히는지
- 표 결과가 해당 조건과 일치하는지
- 추출 필터 표시가 읽기 쉬운지

---

## 4. 설비 데이터 테스트

목적:
- 설비 데이터가 제조 현장 느낌으로 보이는지 확인
- 설비명, 설비ID, 공정, 라인, 가동률, 비가동 사유가 자연스러운지 확인

질문:
- 오늘 설비 가동률 보여줘
- 오늘 DIE_ATTACH 설비 상태 보여줘
- 오늘 WIRE_BOND 설비 상태 보여줘
- 오늘 TEST 설비 상태 보여줘
- 가동률 낮은 설비 순으로 정렬해줘
- 비가동 사유가 있는 설비만 보여줘

확인 포인트:
- 설비ID와 설비명이 자연스러운지
- 가동률과 비가동 사유가 표에 잘 보이는지
- 후속 정렬 요청이 제대로 반영되는지

---

## 5. 불량 데이터 테스트

목적:
- 공정군별 defect taxonomy가 자연스럽게 반영되는지 확인
- 불량률 / 불량수량 / 주요불량유형이 일관성 있게 보이는지 확인

질문:
- 오늘 불량 보여줘
- 오늘 DIE_ATTACH 불량 보여줘
- 오늘 WIRE_BOND 불량 보여줘
- 오늘 FLIP_CHIP 불량 보여줘
- 오늘 TEST 불량 보여줘
- 불량률 높은 순으로 정렬해줘
- 주요불량유형별로 그룹화해줘
- 공정군별 평균 불량률 보여줘

확인 포인트:
- `주요불량유형`이 공정군에 맞는 패턴인지
- 불량률 정렬과 그룹화가 잘 되는지
- 텍스트 응답도 후처리된 결과 기준인지

---

## 6. WIP 데이터 테스트

목적:
- WIP 상태와 대기 흐름이 자연스럽게 보이는지 확인
- 상태 필터링과 후속 분석이 잘 동작하는지 확인

질문:
- 오늘 WIP 보여줘
- 오늘 TEST 공정 WIP 보여줘
- 오늘 PACK_OUT 공정 WIP 보여줘
- HOLD 상태 WIP만 보여줘
- REWORK 상태 WIP만 보여줘
- WAIT_QA 상태 WIP만 보여줘
- 재공수량 상위 10개만 보여줘
- 공정군별 재공수량 합계 보여줘

확인 포인트:
- `상태`, `재공수량`, `avg_wait_minutes`가 자연스러운지
- HOLD / REWORK / WAIT 계열 상태가 잘 필터링되는지
- 후속 그룹화가 잘 동작하는지

---

## 7. 파라미터 승계 테스트

목적:
- 첫 조회 이후 날짜나 조건이 다음 질문으로 자연스럽게 이어지는지 확인

시나리오 A:
- 오늘 DIE_ATTACH 생산량 보여줘
- 불량도 보여줘

시나리오 B:
- 오늘 LPDDR5X_8533 생산량 보여줘
- 불량 보여줘

시나리오 C:
- 오늘 TEST 공정 WIP 보여줘
- 설비 상태도 보여줘

확인 포인트:
- 두 번째 질문에서 날짜가 자동 승계되는지
- 필요한 경우 공정 / 제품 조건도 적절히 이어지는지
- 컨텍스트 표시가 실제 질문 흐름과 맞는지

---

## 8. 단순 후속 pandas 분석 테스트

목적:
- `current_data` 기반 후속 분석이 가장 기본적인 패턴에서 잘 작동하는지 확인

시나리오:
- 오늘 생산량 보여줘
- 상위 5개만 보여줘
- 하위 5개만 보여줘
- MODE별로 그룹화해줘
- 공정군별로 그룹화해줘
- 라인별로 그룹화해줘
- 생산량 높은 순으로 정렬해줘

확인 포인트:
- 결과 테이블이 실제로 변형되는지
- 분석 요약과 생성된 pandas 코드가 보이는지
- 텍스트 응답도 변형된 결과 기준인지

---

## 9. 복합 후속 pandas 분석 테스트

목적:
- 그룹화 + 정렬 + 상위 N 같은 복합 질의가 잘 되는지 확인

시나리오:
- 오늘 생산량 보여줘
- MODE별로 그룹화하고 상위 3개만 보여줘
- MODE, DEN별로 그룹화하고 상위 5개만 보여줘
- MODE, DEN, TECH별로 그룹화해주고 상위 10개 보여줘
- 공정군별 생산량 합계로 상위 5개 보여줘
- 라인별 평균 생산량 보여줘
- TECH별로 그룹화한 뒤 생산량 높은 순으로 정렬해줘

확인 포인트:
- 그룹 기준 컬럼이 정확히 잡히는지
- 집계 후 정렬이 제대로 반영되는지
- 결과 컬럼과 생성 코드가 일치하는지

---

## 10. 그룹별 상위 N 테스트

목적:
- 가장 중요한 후속 분석 시나리오 중 하나인 `그룹별 top N`이 잘 처리되는지 확인

시나리오:
- 오늘 생산량 보여줘
- MODE별로 상위 3개씩만 보여줘
- 공정군별로 상위 2개씩만 보여줘
- MODE, DEN별로 그룹화해주고 그룹별로 상위 3개씩만 보여줘
- TECH별로 그룹화한 다음 그룹별 상위 2개만 보여줘

확인 포인트:
- 단순 global top N이 아니라 그룹별 top N으로 해석되는지
- 결과 행 수가 기대와 크게 다르지 않은지
- 분석 요약에 그룹화와 상위 N 추출이 잘 드러나는지

---

## 11. 필터링 후 재분석 테스트

목적:
- 후속 질문에서 현재 결과를 좁힌 뒤 다시 분석하는 흐름이 되는지 확인

시나리오:
- 오늘 생산량 보여줘
- LPDDR5X_8533만 보여줘
- 공정군별로 그룹화해줘

시나리오:
- 오늘 불량 보여줘
- WIRE_BOND만 보여줘
- 불량률 높은 순으로 정렬해줘

시나리오:
- 오늘 WIP 보여줘
- HOLD 상태만 보여줘
- 공정군별 재공수량 합계 보여줘

확인 포인트:
- 중간 필터 요청이 제대로 반영되는지
- 그 다음 분석이 필터된 결과 기준으로 진행되는지

---

## 12. 현업형 탐색 질문 테스트

목적:
- 실제 제조 현장에서 할 법한 질문이 어느 정도 자연스럽게 처리되는지 확인

질문:
- 어떤 WIRE_BOND 공정에서 불량률이 가장 높아?
- LPDDR5X_8533에서 defect_rate 높은 순으로 상위 5개 보여줘
- TEST 공정군 기준으로 공정별 생산량 요약해줘
- 가동률 저하가 큰 설비만 보여줘
- HOLD 상태 WIP가 많은 공정군 보여줘
- FC 기술 제품만 따로 정리해줘
- PACK_OUT 관련 제품만 보여줘

확인 포인트:
- 완전 정형 문장이 아니어도 적절히 해석되는지
- 지원 불가 대신 현재 결과 기준 분석으로 유도되는지

---

## 13. 오류 및 예외 메시지 테스트

목적:
- 애매한 질문이나 누락 질문에서 안내가 친절한지 확인

질문:
- 보여줘
- 분석해줘
- 상위 3개만 보여줘
- 어제꺼
- MODE별로 정리해줘

확인 포인트:
- 날짜가 없을 때 날짜 재질문이 나오는지
- 조회 대상이 없을 때 생산/목표/불량/설비/WIP 중 하나를 다시 말해달라고 안내하는지
- 현재 데이터가 없는 상태에서 후속 분석 요청 시 먼저 원본 조회를 하라고 안내하는지

---

- 오늘 WIP 보여줘
