# Architecture

## 목적
제조 에이전트 구현에서 공통으로 유지해야 할 핵심 구조를 정의합니다.

## 공통 아키텍처
1. 질문 해석
2. 필수 파라미터 추출
3. 필요한 데이터셋 선택
4. 원본 데이터 조회
5. 현재 데이터 기반 후속 분석
6. 답변 생성

## 하네스가 강제할 것
- 질문 해석과 도메인 지식은 리포지터리 내부 문서로 확인 가능해야 함
- 데이터셋 이름, 컬럼 의미, 분석 패턴은 문서화되어야 함
- 후속 분석은 현재 결과(`current_data` 또는 동등 개념)를 기준으로 해야 함
- 안전 실행 규칙이 있어야 함
- 평가 질문 세트가 존재해야 함

## 구현 리포지터리 차이 허용 범위
- LangGraph 사용 여부
- Streamlit/UI 포함 여부
- 내부 파일명과 함수명

## 구현 리포지터리 차이 비허용 범위
- 도메인 용어 해석 기준
- 데이터셋 정의 의미
- 달성율/불량률 같은 핵심 지표 해석
