# 테스트 질문 모음

이 문서는 운영자나 개발자가 기능을 빠르게 점검할 수 있도록 만든 질문 모음입니다.

각 질문은 아래를 확인하기 위해 사용합니다.

- 어떤 데이터셋이 선택되는지
- 도메인 조건이 잘 추출되는지
- 후속 pandas 분석이 제대로 되는지
- para 승계가 자연스러운지

## 1. 기본 조회

### 질문

- `오늘 ASSY_PREP 공정 생산량 보여줘`
- `오늘 TEST 공정 WIP 보여줘`
- `오늘 LPDDR5X_8533 수율 보여줘`
- `오늘 DIE_ATTACH 스크랩 보여줘`

### 기대 포인트

- 날짜, 공정군, 제품 조건이 잘 반영됨
- 결과 표 컬럼이 데이터셋 성격에 맞음

## 2. 여러 데이터셋 조회

### 질문

- `오늘 생산과 목표를 같이 보여줘`
- `오늘 생산과 불량을 같이 보여줘`
- `오늘 설비와 WIP를 같이 보여줘`

### 기대 포인트

- 여러 retrieval tool이 함께 선택됨
- 응답이 여러 데이터셋을 준비했다는 흐름으로 나옴

## 3. 계산 질문

### 질문

- `오늘 생산량과 목표 데이터를 조회해줘 그리고 공정별로 목표 대비 생산 달성율을 알려줘`
- `오늘 생산과 불량 데이터를 같이 조회해줘 그리고 공정군별 평균 불량률과 생산량을 같이 보여줘`
- `오늘 생산과 목표와 불량을 같이 조회해줘 그리고 MODE별로 목표 대비 실적과 평균 불량률을 보여줘`

### 기대 포인트

- 여러 데이터셋 조회 뒤 pandas 계산이 이어짐
- `달성율`, `평균 불량률` 같은 파생 컬럼이 실제로 생성됨

## 4. 후속 분석

### 질문 흐름

1. `오늘 생산량 보여줘`
2. `상위 5개만 보여줘`
3. `공정군별로 그룹화해줘`
4. `생산량 높은 순으로 정렬해줘`

### 기대 포인트

- 2번 이후는 새 조회가 아니라 후속 분석으로 처리됨
- 생성된 pandas 코드가 질문과 맞음

## 5. 복합 후속 분석

### 질문 흐름

1. `오늘 생산량 보여줘`
2. `mode, den, tech별로 그룹화해주고 그룹별로 상위 3개씩만 보여줘`
3. `LPDDR5X_8533만 남기고 공정군별로 요약해줘`

### 기대 포인트

- group by + filter + top N 조합이 자연스럽게 처리됨

## 6. para 승계

### 질문 흐름

1. `오늘 LPDDR5X_8533 생산량 보여줘`
2. `불량 보여줘`
3. `WIP도 보여줘`
4. `설비 상태도 보여줘`

### 기대 포인트

- 날짜와 제품 조건이 다음 질문에도 이어짐

## 7. 홀드 / 대기시간 / 대표값

### 질문

- `MODE별로 hold lot 수와 대표 hold 사유를 보여줘`
- `MODE별로 평균 대기시간과 hold lot 수를 같이 정리해줘`
- `공정군별로 재공수량 합계와 대표 상태를 보여줘`

### 기대 포인트

- `hold_reason`, `avg_wait_minutes`, `상태`가 올바른 컬럼으로 사용됨

## 8. 없는 컬럼 요청

### 질문 흐름

1. `오늘 목표 보여줘`
2. `공정군별로 그룹화해줘`
3. `담당자별로 정리해줘`

### 기대 포인트

- 앱이 깨지지 않음
- `현재 결과 테이블에 없습니다` 안내가 나옴

## 9. 새 데이터셋 점검

### 질문

- `오늘 수율 보여줘`
- `오늘 LPDDR5X_8533 홀드 lot 보여줘`
- `오늘 FLIP_CHIP 레시피 조건 보여줘`
- `오늘 lot 이력 보여줘`

### 기대 포인트

- 새 데이터셋도 기존 흐름과 같은 방식으로 조회됨

## 10. 빠른 스모크 테스트

시간이 없으면 아래만 먼저 실행해도 됩니다.

- `오늘 ASSY_PREP 공정 생산량 보여줘`
- `오늘 생산량과 목표 데이터를 조회해줘 그리고 공정별로 목표 대비 생산 달성율을 알려줘`
- `오늘 생산량 보여줘`
- `mode, den, tech별로 그룹화해주고 그룹별로 상위 3개씩만 보여줘`
- `MODE별로 hold lot 수와 대표 hold 사유를 보여줘`
- `MODE별로 평균 대기시간과 hold lot 수를 같이 정리해줘`
