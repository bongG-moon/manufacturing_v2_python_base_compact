# 테스트 질문 시나리오

이 문서는 `compact_manufacturing_service`의 핵심 기능을 실제 사용자 질문 흐름처럼 점검하기 위한 예시 모음입니다.

이번 버전은 단순 조회보다 아래를 더 중점적으로 확인하도록 구성했습니다.

- 여러 데이터셋을 한 번에 조회하는 질문
- 도메인 정보 기반 조건 추출
- `current_data` 기반 후속 pandas 분석
- 첫 질문에서 잡힌 para가 다음 질문에도 이어지는지
- 다른 데이터 주제로 넘어가도 날짜 / 공정 / 제품 조건이 자연스럽게 연결되는지

## 권장 테스트 순서

1. 단일 조회와 기본 도메인 추출 확인
2. 복수 데이터셋 조회 확인
3. 복수 데이터셋 + 계산 질문 확인
4. 후속 pandas 분석 확인
5. para 승계와 주제 전환 확인
6. 없는 컬럼 요청 안내 확인

---

## 1. 기본 도메인 추출 테스트

목적:
- 날짜, 공정군, MODE, DEN, TECH, MCP 조건이 잘 추출되는지 확인
- 결과 표와 필터 표시가 질문과 맞는지 확인

질문:
- 오늘 ASSY_PREP 공정 생산량 보여줘
- 오늘 WIRE_BOND 공정 불량 보여줘
- 오늘 TEST 공정 WIP 보여줘
- 오늘 LPDDR5X_8533 생산량 보여줘
- 오늘 64Gb FC 제품 불량 보여줘
- 오늘 MCP-LP5X-064G-4H 제품 생산량 보여줘
- 오늘 UFS3.1 제품 WIP 보여줘

확인 포인트:
- 날짜가 `오늘` 기준으로 잘 해석되는지
- `process_name`, `product_name`, `mode`, `den`, `tech`, `mcp_no`가 적절히 추출되는지
- 결과 테이블이 실제로 질문 조건에 맞는 행만 보여주는지
- 추출된 필터 조건이 UI에 확인 가능하게 보이는지

---

## 2. 공정군과 실제 공정 단계 연결 테스트

목적:
- 공정군 질문이 실제 공정 단계 데이터로 자연스럽게 이어지는지 확인
- 제조 도메인 사전이 결과에 반영되는지 확인

질문:
- 오늘 DIE_ATTACH 공정 생산량 보여줘
- 오늘 FLIP_CHIP 공정 불량 보여줘
- 오늘 PACK_OUT 공정 WIP 보여줘
- 오늘 MOLD 공정 설비 상태 보여줘

확인 포인트:
- 결과 테이블의 `공정군`이 질문과 일치하는지
- `공정` 컬럼에 그 공정군에 맞는 실제 단계가 나오는지
- 설비 / 불량 / WIP 같은 다른 데이터셋에서도 같은 공정군 조건이 일관되게 적용되는지

---

## 3. 복수 데이터셋 동시 조회 테스트

목적:
- 질문 안에 여러 데이터 주제가 있으면 여러 retrieval tool이 함께 호출되는지 확인
- `current_datasets` 구조가 잘 생성되는지 확인

질문:
- 오늘 생산과 목표를 같이 보여줘
- 오늘 생산과 불량을 같이 보여줘
- 오늘 설비와 WIP를 같이 보여줘
- 오늘 생산과 목표와 불량을 같이 보여줘

확인 포인트:
- `tool_results`에 여러 retrieval 결과가 들어오는지
- 현재 결과가 단일 표가 아니라 여러 데이터셋 개요로 보이는지
- 내부적으로 `current_datasets`가 채워지는지
- 응답 텍스트가 “여러 데이터셋을 준비했다”는 흐름으로 자연스럽게 나오는지

---

## 4. 복수 데이터셋 + 계산 질문 테스트

목적:
- 단순 다중 조회가 아니라, 여러 데이터셋을 함께 가져온 뒤 pandas 분석까지 이어지는지 확인
- 도메인 조건이 유지된 상태에서 계산 질문이 자연스럽게 되는지 확인

질문:
- 오늘 생산량과 목표 데이터를 조회해줘 그리고 공정별로 목표 대비 생산량 달성율을 알려줘
- 오늘 생산과 불량 데이터를 같이 조회해줘 그리고 공정군별 평균 불량률과 생산량을 같이 보여줘
- 오늘 생산과 목표와 불량을 같이 조회해줘 그리고 MODE별로 목표 대비 실적과 평균 불량률을 보여줘
- 오늘 TEST 공정 생산, 불량, WIP를 같이 조회해줘 그리고 공정별로 비교해줘

확인 포인트:
- 질문 안의 여러 tool이 같이 호출되는지
- 분석 전에 필요한 공통 분석용 테이블이 생성되는지
- LLM이 새 계산 컬럼을 만들더라도 정상 실행되는지
- 최종 텍스트 응답이 “계산된 결과 테이블” 기준으로 설명되는지

---

## 5. current_data 기반 후속 분석 테스트

목적:
- 첫 조회 이후 후속 질문을 새 조회가 아니라 현재 테이블 가공으로 처리하는지 확인
- LLM이 pandas 코드를 직접 생성해서 적용하는지 확인

질문 시나리오:
1. 오늘 생산량 보여줘
2. 상위 5개만 보여줘
3. MODE별로 그룹화해줘
4. 생산량 높은 순으로 정렬해줘
5. 하위 3개만 보여줘

확인 포인트:
- 2번 이후에는 `analyze_current_data` 경로로 처리되는지
- 생성된 pandas 코드가 질문 의도와 맞는지
- 결과 표와 응답 텍스트가 같은 결과 기준인지

---

## 6. 복합 후속 분석 테스트

목적:
- 여러 단계가 섞인 후속 질문도 정형 rule보다 pandas 코드로 자연스럽게 처리되는지 확인

질문 시나리오:
1. 오늘 생산량 보여줘
2. mode, den, tech별로 그룹화해주고 그룹별로 상위 3개씩만 보여줘
3. LPDDR5X_8533만 남기고 공정군별로 요약해줘
4. 그 결과를 생산량 높은 순으로 다시 정렬해줘

확인 포인트:
- group by + filter + sort + top N 조합이 자연스럽게 되는지
- 질문마다 새 조회가 아니라 현재 결과를 기반으로 이어지는지
- 텍스트 응답도 최종 결과 표 기준으로 설명하는지

---

## 7. para 승계 테스트

목적:
- 첫 질문에서 잡힌 날짜 / 제품 / 공정 조건이 다음 데이터셋으로 넘어가도 유지되는지 확인

질문 시나리오:
1. 오늘 LPDDR5X_8533 생산량 보여줘
2. 불량 보여줘
3. WIP 보여줘
4. 설비 상태도 보여줘

확인 포인트:
- `오늘`, `LPDDR5X_8533` 조건이 후속 질문에도 이어지는지
- 데이터셋이 바뀌어도 주요 para가 유지되는지
- 사용자가 조건을 다시 안 써도 자연스럽게 연결되는지

---

## 8. 공정 중심 주제 전환 테스트

목적:
- 공정 조건을 잡은 뒤 생산 -> 불량 -> 설비 -> WIP로 넘어가도 흐름이 유지되는지 확인

질문 시나리오:
1. 오늘 TEST 공정 생산량 보여줘
2. 불량 보여줘
3. 설비 상태도 보여줘
4. WIP도 보여줘

확인 포인트:
- `TEST 공정` 조건이 계속 이어지는지
- 다른 데이터셋에서도 공정 조건이 제대로 반영되는지
- 응답이 억지스럽지 않고 자연스럽게 이어지는지

---

## 9. 설비 / WIP 도메인 연결 테스트

목적:
- 현업형 운영 질문에서 설비와 WIP를 함께 볼 수 있는지 확인
- 도메인 용어가 자연스럽게 반영되는지 확인

질문:
- 오늘 TEST 공정 WIP 보여줘 그리고 설비 상태도 같이 보여줘
- 오늘 WIRE_BOND 설비 상태와 WIP를 같이 보여줘
- 오늘 MOLD 설비 가동률과 WIP 상태를 같이 보여줘

확인 포인트:
- 설비와 WIP가 각각 적절한 tool로 조회되는지
- 질문이 한 번에 여러 데이터를 요구해도 무리 없이 처리되는지
- 결과 설명이 “가동률”과 “재공/홀드 상태”를 함께 언급하는지

---

## 10. 품질 중심 복합 질문 테스트

목적:
- 품질 관점 질문에서 생산 / 불량 / 제품 조건이 함께 잘 반영되는지 확인

질문:
- 오늘 LPDDR5X_8533 생산과 불량을 같이 조회해줘 그리고 공정군별로 비교해줘
- 오늘 DDR5_6400 생산과 불량을 같이 보여주고 불량률 높은 공정 순으로 정리해줘
- 오늘 FC 기술 제품의 생산과 불량을 같이 보여주고 주요 불량 유형도 같이 알려줘

확인 포인트:
- 제품 / 기술 조건이 여러 데이터셋에 같이 적용되는지
- 비교 질문이 새 계산 결과로 이어지는지
- 응답이 생산량과 불량률을 섞어서 잘 설명하는지

---

## 11. 없는 컬럼 요청 안내 테스트

목적:
- 현재 결과에 없는 기준을 요청했을 때 에러가 아니라 안내 문구로 처리되는지 확인

질문 시나리오:
1. 오늘 목표 보여줘
2. 공정군별로 그룹화해줘
3. MODE별로 그룹화해줘

추가 시나리오:
1. 오늘 생산량 보여줘
2. 담당자별로 정리해줘

확인 포인트:
- `현재 결과 테이블에 없습니다` 형태로 안내되는지
- 사용 가능한 주요 컬럼이 함께 제시되는지
- 실패하더라도 앱 전체 흐름은 깨지지 않는지

---

## 12. 새 조회 vs 후속 분석 구분 테스트

목적:
- 현재 테이블이 있어도, 사용자가 새 데이터를 요청하면 새 조회로 correctly 분기하는지 확인

질문 시나리오:
1. 오늘 생산량 보여줘
2. 상위 5개만 보여줘
3. 오늘 생산량과 목표 데이터를 조회해줘 그리고 공정별로 목표 대비 생산량 달성율을 알려줘

확인 포인트:
- 2번은 후속 분석으로 처리되는지
- 3번은 이전 top 5 테이블이 아니라 새 조회로 처리되는지
- 다중 조회 + 계산 질문이 잘 동작하는지

---

## 13. 신규 데이터셋 개별 조회 테스트

목적:
- 새로 추가한 `수율 / 홀드 / 스크랩 / 레시피 / LOT 이력` 데이터가 각각 잘 조회되는지 확인
- 기존 공정 / 제품 / 기술 조건이 새 데이터셋에도 동일하게 반영되는지 확인

질문:
- 오늘 수율 보여줘
- 오늘 TEST 공정 수율 보여줘
- 오늘 LPDDR5X_8533 홀드 lot 보여줘
- 오늘 DIE_ATTACH 스크랩 보여줘
- 오늘 FLIP_CHIP 레시피 조건 보여줘
- 오늘 lot 이력 보여줘
- 오늘 WIRE_BOND 공정 lot 이력 보여줘

확인 포인트:
- 새 데이터셋도 기존 retrieval 구조 안에서 정상 조회되는지
- 공정 / 제품 / MODE / TECH 조건이 새 데이터셋에도 동일하게 먹는지
- 수율은 `tested_qty`, `pass_qty`, `yield_rate`가 자연스럽게 보이는지
- 홀드는 `hold_reason`, `hold_owner`, `hold_hours`가 보이는지
- 스크랩은 `scrap_qty`, `scrap_rate`, `loss_cost_usd`가 보이는지
- 레시피는 `recipe_id`, `temp_c`, `pressure_kpa`, `process_time_sec`가 보이는지
- LOT 이력은 `lot_id`, `route_step`, `current_status`, `elapsed_hours`가 보이는지

---

## 14. 신규 데이터셋 복합 분석 테스트

목적:
- 새 데이터셋이 기존 생산 / 목표 / 불량 / 설비 / WIP 데이터와 함께 자연스럽게 엮이는지 확인
- multi-tool 호출과 계산 질문이 더 넓은 범위에서 잘 동작하는지 확인

질문:
- 오늘 생산과 수율을 같이 조회해줘 그리고 공정군별로 비교해줘
- 오늘 불량과 스크랩을 같이 조회해줘 그리고 공정별 손실 큰 순으로 보여줘
- 오늘 WIP와 홀드 lot을 같이 보여주고 hold 시간이 긴 순으로 정리해줘
- 오늘 TEST 공정 수율과 불량을 같이 조회해줘 그리고 dominant_fail_bin 기준으로 설명해줘
- 오늘 FLIP_CHIP 공정 레시피와 불량을 같이 조회해줘 그리고 수율과 함께 비교해줘
- 오늘 LPDDR5X_8533 생산과 lot 이력을 같이 보여줘 그리고 어느 공정에서 오래 머무는지 알려줘

확인 포인트:
- 새 데이터셋과 기존 데이터셋이 함께 호출되는지
- 공통 분석용 테이블을 만들 수 있는 조합은 계산까지 이어지는지
- 공통 테이블이 어려운 조합은 무리하게 merge하지 않고도 자연스럽게 안내되는지
- 응답이 실제 계산된 결과 또는 준비된 데이터셋 기준으로 설명되는지

---

## 15. 새 데이터셋 para 승계 테스트

목적:
- 새 데이터셋으로 주제가 넘어가도 날짜 / 공정 / 제품 조건이 이어지는지 확인

질문 시나리오:
1. 오늘 LPDDR5X_8533 생산량 보여줘
2. 수율 보여줘
3. 홀드 lot도 보여줘
4. 스크랩도 보여줘

추가 시나리오:
1. 오늘 TEST 공정 수율 보여줘
2. 불량 보여줘
3. 레시피 조건도 보여줘
4. lot 이력도 보여줘

확인 포인트:
