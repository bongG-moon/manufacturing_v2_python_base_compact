# 질문 사용법

이 문서는 사용자가 어떤 식으로 질문하면 좋은지, 시스템이 어떻게 이해하는지를 설명합니다.

## 1. 질문은 두 종류로 나뉩니다

### 새 조회 질문

예:

- `오늘 생산량 보여줘`
- `오늘 TEST 공정 WIP 보여줘`
- `오늘 LPDDR5X_8533 수율 보여줘`

이런 질문은 새 데이터를 가져옵니다.

### 후속 분석 질문

예:

- `상위 5개만 보여줘`
- `공정군별로 그룹화해줘`
- `목표 대비 달성율을 계산해줘`

이런 질문은 방금 본 결과 표를 다시 분석합니다.

## 2. current_data가 무엇인가

`current_data`는 방금 사용자에게 보여준 결과 표입니다.

후속 질문은 보통 이 `current_data`를 기준으로 처리됩니다.

예:

1. `오늘 생산량 보여줘`
2. `MODE별로 정리해줘`
3. `상위 3개만 보여줘`

2번과 3번은 새 조회가 아니라 `current_data`를 가공하는 요청입니다.

이 값은 코드에서 아래 흐름으로 사용됩니다.

- `app.py`의 `_run_chat_turn()`
  - `run_agent()`에 현재 표 전달
- `core/agent.py`의 `_has_current_data()`
  - 현재 표가 있는지 확인
- `core/agent.py`의 `_choose_query_mode()`
  - 후속 분석으로 갈지 결정
- `core/data_analysis_engine.py`의 `execute_analysis_query()`
  - 실제 pandas 분석 수행

## 3. para 승계가 무엇인가

첫 질문에서 잡힌 조건이 다음 질문에도 이어지는 기능입니다.

예:

1. `오늘 LPDDR5X_8533 생산량 보여줘`
2. `불량 보여줘`
3. `WIP도 보여줘`

이 경우 `오늘`, `LPDDR5X_8533` 조건이 다음 데이터셋에도 이어질 수 있습니다.

## 4. 좋은 질문 예시

### 단일 조회

- `오늘 ASSY_PREP 공정 생산량 보여줘`
- `오늘 TEST 공정 설비 상태 보여줘`

### 여러 데이터셋 조회

- `오늘 생산과 목표를 같이 보여줘`
- `오늘 생산과 불량을 같이 보여줘`

### 계산 질문

- `공정별로 목표 대비 생산 달성율을 알려줘`
- `공정군별 평균 불량률과 최빈 불량유형을 정리해줘`

### 후속 분석 질문

- `상위 5개만 보여줘`
- `공정군별로 그룹화해줘`
- `MODE별로 평균 대기시간과 hold lot 수를 같이 정리해줘`

## 5. 시스템이 헷갈리기 쉬운 표현

아래 표현은 가능하면 조금 더 구체적으로 말하면 좋습니다.

- `좋은 공정만 보여줘`
- `문제 있는 것만 보여줘`
- `비슷한 것끼리 정리해줘`

대신 이렇게 말하는 편이 좋습니다.

- `불량률 높은 공정만 보여줘`
- `상태가 HOLD인 것만 보여줘`
- `공정군별로 그룹화해줘`

## 6. 오류가 나면 먼저 확인할 것

### 없는 컬럼 요청

예:

- `담당자별로 정리해줘`

현재 표에 `담당자` 컬럼이 없으면 안내 문구가 나옵니다.

### 새 조회가 필요한데 후속 분석처럼 말한 경우

예:

- 이전에 WIP를 보고 있었는데 `불량 보여줘`를 했을 때

이 질문이 새 조회인지 후속 분석인지 애매할 수 있습니다.  
이럴 때는 `오늘 TEST 공정 불량 보여줘`처럼 다시 말하면 더 안정적입니다.

## 7. 질문이 코드로 처리되는 방식

초보자 입장에서는 “질문이 어디서 처리되는가”가 중요합니다.

### 새 조회 질문

예: `오늘 생산량 보여줘`

코드 흐름:

1. `core/parameter_resolver.py`의 `resolve_required_params()`
2. `core/agent.py`의 `_run_retrieval()`
3. `core/data_tools.py`의 `pick_retrieval_tools()`
4. `core/data_tools.py`의 `execute_retrieval_tools()`

### 후속 분석 질문

예: `상위 5개만 보여줘`

코드 흐름:

1. `core/agent.py`의 `_run_followup_analysis()`
2. `core/data_analysis_engine.py`의 `execute_analysis_query()`
3. `core/analysis_llm.py`의 `build_llm_plan()`
4. `core/safe_code_executor.py`의 `execute_safe_dataframe_code()`

즉 후속 질문은 “LLM이 pandas 코드를 짠 뒤 실행하는 구조”입니다.
