# 컬럼명 규칙 안내

이 프로젝트는 현재 “내부 계산용 컬럼”과 “현업에서 읽기 쉬운 컬럼”이 일부 섞여 있습니다.

예:
- 영문 컬럼: `production`, `target`, `inspection_qty`, `defect_rate`
- 한글 컬럼: `공정`, `공정군`, `라인`, `불량수량`, `재공수량`, `설비명`

## 왜 이렇게 되어 있나

- pandas 집계나 LLM 코드 생성에는 영문 컬럼이 편한 경우가 있습니다
- 실제 화면과 현업 질문에는 한글 컬럼이 읽기 쉽습니다

그래서 현재는 혼합 구조를 사용하고 있습니다.

## 현재 규칙

- 제조 식별용 컬럼은 한글 위주
  - `공정`
  - `공정군`
  - `라인`
  - `설비명`
  - `불량수량`
  - `재공수량`

- 계산/집계에 자주 쓰는 컬럼은 영문 위주
  - `production`
  - `target`
  - `inspection_qty`
  - `defect_rate`
  - `planned_hours`
  - `actual_hours`

- 제품 관련 표준 키는 원문 유지
  - `MODE`
  - `DEN`
  - `TECH`
  - `LEAD`
  - `MCP_NO`

## 화면 표시 규칙

- 수량 컬럼은 [core/number_format.py](/C:/Users/qkekt/Desktop/compact_manufacturing_service/core/number_format.py) 에서 자동 감지합니다
- 큰 수량 컬럼은 헤더가 `production (K)`, `재공수량 (M)`처럼 보일 수 있습니다
- 이 변경은 화면 표시용이며, 내부 원본 컬럼명은 그대로 유지됩니다

## 초보자에게 추천하는 수정 원칙

- 새로운 컬럼을 추가할 때는 먼저 “이 값이 식별용인지 계산용인지”를 정합니다
- 식별용이면 한글 컬럼으로 추가해도 좋습니다
- 계산/집계 중심이면 영문 컬럼으로 두는 편이 pandas 코드 작성이 쉽습니다
- 다만 같은 성격의 컬럼은 기존 패턴을 따라가야 혼란이 줄어듭니다

## 나중에 더 단순화하고 싶다면

가장 쉬운 방향은 둘 중 하나입니다.

1. 내부도 전부 한글 컬럼으로 통일
2. 내부는 전부 영문, 화면에서만 한글 라벨로 변환

현재는 2번으로 가기 전 단계의 혼합형 구조라고 보면 됩니다.
